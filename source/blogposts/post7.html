An Explanation of
CSS Media Queries and How to Use Them

OCTOBER 3, 2018

To incorporate media queries into your webpage, you first need to insert a self-closing <meta> within the <head> of your HTML file, like in the example below:

<!DOCTYPE html>
<html lang='en'>
<meta charset='UTF-8'>
<head>
<tite>...</title>
<meta name="viewport" content="initial-scale:1.0" width="device-width"/>
</head>

Once you have completed inserting that into your <head>, you need to open up your CSS Stylesheet. The important thing at this point is to understand the difference between @media (min-width: ) and @media (max-width: ) in relation to varying screen sizes between mobile, tablet, and desktop.

It should be noted that when using @media (min-width: ), you are referring to any pixel size above that threshold set in pixels, where as @media (max-width: ) refers to anything below that threshold. It is also recommended that you insert your media queries into your CSS sheet as opposed to doing it in your HTML, as this leads to cleaner, more readable code that it easy to understand.

To determine what your threshold should be, a good place to start might be referencing some common device screen sizes, such as 360 px (mobile), 768 px (tablet), and 1200px (desktop). However, it’s always best to resize your browser to see what happens because the rearrangement of any type of content on the page (i.e. images, text, colors, boxes) between those sizes can potentially look incorrect or rearrange themselves in an unexpected manner that you didn’t intend upon. It should also be noted that em/rem is usually reserved for typography sizes, whereas percentages are best applied to your website’s grid and layout.
